# Entropiq — Технические требования

> Персональный финансовый помощник с учётом инфляции и системой прогнозирования финансовых целей

**Версия документа:** 1.0
**Дата:** 2026-02-08
**Статус:** Утверждён

---

## 1. Общее описание

Веб-сервис для учёта личных и семейных финансов с функцией пересчёта покупательной способности с учётом инфляции и AI-рекомендациями для достижения финансовых целей. Freemium-модель монетизации.

**Целевая аудитория:** резиденты РФ (молодые профессионалы 25–35 лет, семьи с детьми 30–45 лет).

**Рабочее название:** Entropiq
**Расположение проекта:** `~/domains/hse/entropiq`
**Репозиторий:** GitHub (приватный)

---

## 2. Архитектура

### 2.1. Тип архитектуры

Монолитное приложение с модульной внутренней структурой (Service classes, Action classes, Events/Listeners). Модульность позволяет при необходимости выделить отдельные сервисы в будущем.

### 2.2. Основные модули

| Модуль | Описание |
|--------|----------|
| **Auth** | Регистрация, аутентификация (SMS), авторизация |
| **Transactions** | Учёт доходов и расходов, категории, повторяющиеся операции |
| **Goals** | Финансовые цели, прогресс, прогнозирование |
| **Inflation** | Интеграция с ЕМИСС, расчёт ИПЦ, пересчёт покупательной способности |
| **AI** | Анализ паттернов, генерация рекомендаций, сценарное моделирование |
| **Reports** | Аналитика, графики, экспорт PDF/Excel |
| **Subscriptions** | Управление подписками, платежи, freemium-логика |
| **Notifications** | Email, SMS, Telegram, push-уведомления |
| **Users** | Профили, настройки, мультиязычность, будущие семейные аккаунты |

### 2.3. API

- REST API с версионированием: `/api/v1/...`
- API проектируется с первого дня с расчётом на будущее мобильное приложение
- Документация API: автоматическая генерация (Scribe или Scramble)

| Уведомления | Telegram Bot API | Push-уведомления в Telegram |

## 5. Аутентификация и авторизация

### 5.1. Аутентификация

- **Метод:** номер телефона + одноразовый SMS-код (OTP)
- **Провайдер SMS:** SMS Aero
- **Срок жизни кода:** 5 минут
- **Rate limiting:** не более 3 попыток ввода кода, не более 1 SMS в 60 секунд на номер
- **Будущее расширение:** OAuth (VK, Google), email + пароль

### 5.2. Авторизация

- **Пакет:** Laravel Sanctum
- **Веб-интерфейс:** cookie-based сессии (SPA stateful auth)
- **API (мобильное приложение):** Bearer-токены
- **Управление доступом:** Laravel Policies + Gates
- **Роли:** реализуются при необходимости (admin, user, family_member — на втором этапе)

## 7. CI/CD

### 7.1. CI — GitHub Actions

Пайплайн запускается на каждый PR:

1. **Lint:** Laravel Pint (проверка код-стайла)
2. **Static Analysis:** Larastan (PHPStan level 6+)
3. **Tests:** Pest (unit + feature)
4. **Build:** проверка сборки Docker-образа

### 7.2. CD — Laravel Envoy

- **Stage:** автоматический деплой при мерже в ветку `stage`
- **Production:** ручной запуск деплоя при мерже в ветку `main`
- **Процесс деплоя:** pull → composer install (--no-dev) → npm build → migrate → cache clear/warm → restart Horizon workers → healthcheck

### 7.3. Git-стратегия ветвления

```
feature/* → PR → stage → PR → main
                  ↓               ↓
            Stage-сервер    Production-сервер

hotfix/*  → PR → main (+ обратный мерж в stage)
```

- **`main`** — production-ready код
- **`stage`** — интеграционная ветка, деплоится на stage-сервер
- **`feature/*`** — функциональные ветки, создаются от `stage`
- **`hotfix/*`** — экстренные исправления, создаются от `main`

## 9. Безопасность

- HTTPS обязателен во всех окружениях (кроме local)
- CSRF-защита для всех форм (Laravel middleware)
- Rate limiting на API-эндпоинты (Laravel Rate Limiter)
- Валидация входных данных через Form Requests
- Хранение секретов через `.env` (не коммитятся в репозиторий)
- Параметризованные запросы (Eloquent ORM — SQL-инъекции исключены)
- Content Security Policy headers
- Audit trail через Spatie Activity Log (логирование всех критичных операций)
- Соответствие 152-ФЗ (обработка персональных данных)

## 10. Бэкапы

- **Метод:** самописный bash-скрипт (`pg_dump` → сжатие → выгрузка на внешнее хранилище)
- **Частота:** каждые 6 часов (cron)
- **Хранение:** внешнее хранилище (S3-совместимое или отдельный сервер)
- **Ретеншн:** 7 дней (28 бэкапов)
- **Будущее:** миграция на Spatie Laravel Backup
